Section 02

Claude Chat

The core product — a conversational AI available on web, desktop, iOS, and Android. This is where most users start, and for many, it's all they'll ever need.

💬
Conversational AI

Claude.ai Chat

A single conversation interface that accesses Claude's full reasoning, writing, coding, and research capabilities — with powerful features built on top.

Core Features

  • Web Search — Claude queries live web results and cites sources. Toggle in Settings.
  • Deep Research — Multi-step research producing long-form, cited reports. Pro+ only.
  • File & Document Upload — Analyze PDFs, DOCX, spreadsheets, images, code files up to 30MB.
  • Artifacts — Live-rendered outputs: React apps, HTML, SVG, Mermaid, Markdown.
  • Code Execution — Claude writes and runs Python. Produces charts, computed results, file processing.
  • Memory — Saves key facts across conversations. View/edit in Settings → Memory.
  • Projects — Persistent workspaces with files, instructions, and memory.
  • Custom Style — Train Claude on your writing voice using samples.
  • User Preferences — Set global tone, format, and expertise-level defaults.
  • Incognito — No-history, no-memory conversations for privacy.
  • Health Data (iOS/Android) — Analyze fitness, sleep, and activity data. US only, Pro+.

Best Use Cases

  • Writing & EditingDrafts, rewrites, ghostwriting, style matching.
  • Research & SummarizationAnalyze long documents; extract and compare insights.
  • Data AnalysisUpload CSVs, generate charts, spot trends.
  • Coding HelpExplain, debug, and write code in any language.
  • BrainstormingExplore ideas, build plans, stress-test arguments.
  • LearningExplain complex topics; generate study guides.
  • Email & CommsDraft messages, adjust tone, plan difficult conversations.
Pro Tip: Projects

Create a Project for any recurring work — a client, a codebase, a newsletter. Upload your brand guidelines, style guide, or reference docs. Set a custom system prompt. Claude will use all of it in every conversation within that project, so you never re-explain context again.

Section 03

Claude Cowork

Cowork brings Claude Code's agentic architecture to knowledge work. Instead of responding one message at a time, Claude can take on complex, multi-step tasks autonomously on your local machine — and come back with finished work.

🖥️
Agentic Desktop · Research Preview

Claude Cowork

An autonomous AI agent running inside Claude Desktop that can read, write, and organize your local files, use your apps via connectors or direct computer use, and complete long-running tasks — with or without you watching.

What Makes Cowork Different from Chat

  • Direct File Access — No uploading. Claude reads and writes files in folders you authorize.
  • Sub-agent Coordination — Breaks complex work into parallel subtasks across multiple internal agents.
  • Scheduled Tasks — Use /schedule to run tasks automatically on a cadence (requires computer awake).
  • Long-running Sessions — No conversation timeouts or context interruptions mid-task.
  • Computer Use — Claude can move your mouse, click, open apps, and navigate the browser on your behalf.
  • Projects in Cowork — Separate contexts per project, each with its own files, instructions, memory, and scheduled tasks.
  • Global Instructions — Set at Settings → Cowork → Edit. Apply to every Cowork session.
  • Plugins — Bundles of Skills, connectors, and sub-agents for specific roles. Available in a marketplace for Team/Enterprise.

What Cowork Excels At

  • File Organization"Sort my Downloads by type, delete anything older than 6 months."
  • Expense Processing"Turn 50 receipt screenshots into a categorized expense spreadsheet."
  • Research Reports"Research our top 5 competitors and produce a structured comparison doc."
  • Morning Briefings"Every weekday at 8am, summarize my overnight emails and calendar."
  • Document Generation"Read last month's data and populate the quarterly report template."
  • Cross-app Workflows"Pull meeting notes from Notion, draft follow-up emails in Gmail."
  • App AutomationVia computer use: navigate any app Claude doesn't have a connector for.
How to Get Started with Cowork
  1. 1

    Install Claude Desktop

    Download from claude.com/download. Available on macOS and Windows. Run the Cowork readiness check first to confirm your machine supports it.

  2. 2

    Open Cowork from the sidebar

    In Claude Desktop, click "Cowork" in the left sidebar to switch from chat mode to agentic task mode.

  3. 3

    Grant folder access

    Tell Claude which folders it can read and write to. This is explicit — Claude only touches what you authorize.

  4. 4

    Set up connectors (optional)

    Connect your tools — Notion, Gmail, Slack, Google Drive, GitHub, and 38+ more — so Claude can work across your apps without screen-scraping.

  5. 5

    Describe your goal and step away

    Tell Claude the outcome you want. Claude shows you its plan and waits for approval before acting. You can monitor in real time or return to finished work.

⚠️ Important Limitation

Cowork tasks only run while Claude Desktop is open and your computer is awake. Scheduled tasks are skipped if the computer is asleep and run once it wakes up. Cowork is also not suitable for HIPAA, FedRAMP, or FSI regulated workloads. Conversation history is stored locally on your machine.

Section 04

Dispatch

Dispatch is the bridge between your phone and your desktop. It turns the Claude mobile app into a remote control for Cowork — so your computer works while you're away from it.

📱
Remote Agent Control

Dispatch

A persistent conversation thread that connects your Claude mobile app (iOS or Android) to Cowork on your desktop. Assign tasks from your phone; your computer executes them in the background.

How Dispatch Works

  • Open Cowork in Claude Desktop, select Dispatch, scan the QR code with your phone. Takes under 2 minutes.
  • Your phone appears as a persistent thread in the Cowork sidebar.
  • Send instructions from your phone → Desktop executes them locally.
  • With computer use enabled, Dispatch can operate any app on your machine.
  • Processing is local and sandboxed — files don't leave your computer.
  • Also available in Claude Code Remote for developer workflows.

Real-World Use Cases

  • While commuting"Summarize all contracts in my Downloads, flag anything expiring before June."
  • In a meeting"Find all files related to Project Alpha and prepare a status summary."
  • On vacation"Check my email every morning and flag anything urgent, send me a digest."
  • Before a meeting"Pull my calendar for tomorrow and create a briefing doc for each appointment."
  • Task monitoringCheck the progress of a long-running Cowork task from your pocket.
⚠️ Security Note

Anthropic's own documentation warns: "Instructions sent remotely trigger real actions including file modification, deletion, and email sending. File damage can happen quickly." Always confirm what Dispatch is about to do before walking away from complex or destructive tasks.

Section 05

Claude Code

Claude Code is a terminal-native agentic coding system. It doesn't just suggest code — it reads your entire codebase, makes multi-file edits, runs tests, monitors CI pipelines, and ships commits. At Anthropic, the majority of code is now written by Claude Code.

⌨️
Agentic Coding System

Claude Code

A command-line tool that operates as an autonomous agent in your development environment. It understands your codebase structure, uses your tools (git, package managers, test runners), and executes multi-step tasks from natural language goals.

What Claude Code Can Do

  • Codebase Understanding — Reads your entire project structure, dependencies, and history using agentic search. No manual context selection.
  • Multi-file Edits — Plans and executes changes across many files to implement a feature or fix a bug correctly.
  • Test Writing & Running — Writes tests, runs them, reads failures, and iterates until they pass.
  • Git Workflows — Handles branches, commits, PRs, and merge conflicts natively.
  • CI/CD Monitoring — Watches GitHub/GitLab pipelines; auto-commits fixes when builds fail.
  • Terminal Tools — Uses any CLI tool you have: npm, pip, kubectl, docker, etc.
  • Computer Use (new) — Can launch and interact with apps during testing workflows.
  • Sub-agent Orchestration — Spawns parallel agents for complex multi-part tasks.
  • IDE Integration — Works natively in VS Code, Cursor, Windsurf, and JetBrains.
  • GitHub @claude — Tag @claude in GitHub comments to trigger agents directly from issues.
  • 1M Token Context — Handles massive monorepos with Sonnet 4.6 / Opus 4.6.

Who Uses Claude Code

  • Senior EngineersOrchestrating multiple agents across large codebases.
  • DevelopersAutomating repetitive tasks: boilerplate, tests, docs, refactors.
  • Non-EngineersProduct managers, founders, and operators building tools by describing goals in plain language.
  • Enterprise TeamsDeploying remotely via Claude Code Remote with SSH, no local install needed.

The Agentic Loop

Claude Code works in three phases per task:
① Gather context → reads files, git state, test output
② Take action → edits files, runs commands
③ Verify results → runs tests, checks output

The loop repeats, chaining dozens of actions, course-correcting as it learns. You can interrupt any time.
Installing Claude Code
  1. 1

    Install via npm

    Run npm install -g @anthropic-ai/claude-code in your terminal. Requires Node.js.

  2. 2

    Navigate to your project

    Run cd your-project then claude. Claude Code automatically reads your project structure.

  3. 3

    Grant permissions

    Claude Code asks before modifying files or running commands. Approve once, or use --auto-approve for non-interactive sessions.

  4. 4

    Extend with MCP and Skills

    Connect MCP servers for external tools (GitHub, Jira, Notion). Install Skills to add domain knowledge and specialized workflows.

Claude Code vs. Chat for Coding

Claude Chat's code help is excellent for explanation, debugging snippets, and writing functions. Claude Code is for when you want Claude to actually execute — run the tests, make the commits, ship the PR. It operates at the project level, not the snippet level.

Section 06

Skills

Skills are the answer to the most common AI frustration: rewriting the same instructions over and over. A Skill packages instructions, scripts, and resources into a reusable module that Claude loads automatically when relevant.

🎯
Reusable Workflow Modules

Claude Skills

Organized folders containing instructions, scripts, and resources that Claude dynamically discovers and loads when they're relevant — and ignores when they're not. Think of them as permanent context for recurring tasks.

How Skills Work

  • Each Skill lives in a folder with a SKILL.md file — YAML metadata at the top, instructions below.
  • Progressive disclosure — Claude scans metadata first (~100 tokens), loads full instructions only when relevant (<5k tokens). Multiple Skills can be available without bloating context.
  • Skills can include executable scripts — for deterministic operations like PDF extraction, data normalization, or file processing.
  • Skills work across Claude Chat, Claude Code, and Office Add-ins — the same skill file works everywhere.
  • Open standard (agentskills.io) — Skills you build also work in Cursor, ChatGPT, and other platforms that adopt the spec.
  • For teams: admins manage org-wide Skills via the plugin marketplace. Versioned, auditable, one-click deployment.
  • Invoke manually with /skill-name or let Claude trigger automatically when the task matches.

Skills vs. Other Features

  • Skills vs. MemoryMemory stores facts about you. Skills encode how to do a task.
  • Skills vs. InstructionsInstructions set tone and style. Skills define step-by-step procedures.
  • Skills vs. MCPMCP connects Claude to tools. Skills tell Claude how to use those tools correctly.
  • Skills vs. PluginsPlugins bundle Skills + connectors + sub-agents into one install. Skills are the building block.

Built-In Starter Skills

  • Excel: Audit formulas, populate DCF/LBO templates, clean data
  • PowerPoint: Build competitive decks, review IB narratives
  • PDF/DOCX/XLSX: Create and manipulate files from Claude Chat
Building Your Own Skill

A Skill is just a folder with a SKILL.md file. No coding required — if you can write documentation, you can build a Skill. Use the built-in Skill Creator in Claude to be guided through it.

Minimal Skill Structure

my-skill/
  SKILL.md  ← Required. YAML metadata + instructions in Markdown.
  templates/  ← Optional supporting files
  scripts/  ← Optional executable code for deterministic steps

The YAML description field is the most important part — Claude uses it to decide when to activate the Skill. Make it specific.

Model Context Protocol (MCP)

MCP is Anthropic's open standard for connecting AI models to external tools and data sources. Donated to the Linux Foundation, it has been adopted by OpenAI, Google DeepMind, and Microsoft — becoming the de facto protocol for AI tool integration.

🔌
Open Protocol

Model Context Protocol

A standardized interface that lets Claude connect to any external service — databases, APIs, SaaS apps, local tools — through lightweight MCP servers. No custom integrations; one protocol for everything.

What MCP Enables

  • Connect Claude Code to 38+ apps natively: Notion, Gmail, Slack, GitHub, Figma, Jira, and more.
  • Query live databases, APIs, and enterprise systems mid-task.
  • In Claude for Excel: pull in S&P Global, LSEG, FactSet, PitchBook data without leaving your spreadsheet.
  • Custom MCP servers let organizations build internal integrations with their own data sources.
  • Tool search keeps context lean — Claude loads only the MCP tools it actually needs for each task.

Example Workflows via MCP

  • Dev workflow"Implement the feature from JIRA-4521, check Sentry for related errors, and create a PR on GitHub."
  • Research"Find the Figma designs posted in Slack and update our email template accordingly."
  • Outreach"Find 10 users from our PostgreSQL DB who used this feature and draft Gmail invites for a feedback call."
  • Finance"Pull LSEG bond data into this Excel model and update the DCF assumptions."
Skills + MCP: The Power Combo

MCP gives Claude access to tools. Skills teach Claude how to use those tools correctly — with your organization's workflows, standards, and conventions baked in. Use both together for production-grade automation.

Section 10

Prompting Mastery

The quality of Claude's output depends heavily on how you ask. These six techniques work across every surface — chat, Cowork, Code, and the API.

1. Be Specific

The most common mistake. Vague prompts produce generic results. Every additional detail constrains the output toward what you actually want.

Too vague
"Help me with a presentation."
Specific
"Create a 10-slide outline for our Q2 sales review. Cover performance vs. targets, top 3 products, 2 customer wins, and Q3 goals. Audience is the executive team — no jargon, lead with numbers."
2. Show an Example

A concrete sample of the style, tone, or format you want is worth a paragraph of instructions. Claude learns from examples instantly.

No reference
"Write a professional email about a project delay."
With example
"Here's a past email I sent: [paste]. Now write a similar one for a 3-week delay caused by supplier issues. Same tone, same sign-off style."
3. Assign a Role

Telling Claude to act as a specific expert dramatically deepens the response — it changes not just the vocabulary but the perspective.

Generic
"Analyze this financial report."
Expert framing
"Act as a CFO preparing a board briefing. Analyze this Q2 report: executive summary, KPIs in a table, top 3 risks, and 2 strategic recommendations. Be direct."
4. Ask for Step-by-Step Reasoning

For complex or analytical tasks, making Claude think out loud improves accuracy and catches errors early.

Skips process
"How should I improve team productivity?"
Structured thinking
"Think step-by-step. First identify the most common blockers, then propose solutions for each, then flag implementation challenges, then suggest how to measure success. Show your reasoning."
5. Iterate Specifically

If the first response isn't right, precise feedback produces far better revisions than "make it better."

Vague feedback
"Make it better."
Precise revision
"Good structure. Now: (1) make the tone warmer, (2) add a specific customer win in paragraph 2, (3) cut paragraph 3 to 2 sentences — it repeats what's already in the intro."
6. Define Output Format

Tell Claude exactly how to structure the response. It will follow it precisely.

Format left open
"Compare these three project management tools."
Format specified
"Compare Asana, Trello, and Notion in a table. Columns: key features, ease of use, scalability, price, best for. Add a 1-sentence recommendation at the end."
Advanced: XML Tags

For complex, structured prompts, use XML-style tags to separate instruction types: wrap context in <context>...</context>, documents in <document>...</document>, and the task in <task>...</task>. This helps Claude parse long inputs with multiple components accurately. See docs.claude.com/en/docs/build-with-claude/prompt-engineering/overview for the full guide.

Just getting started? Begin with Claude Chat. Master prompting, use Projects for recurring work, and try Skills for repeated tasks.

Knowledge worker? Add Cowork and Dispatch once you're comfortable. Start with simple file organization tasks, then move to scheduled automation.
