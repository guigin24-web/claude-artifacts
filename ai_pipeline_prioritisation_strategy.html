Tab 3 of 4 · Change management
5 Problems → Root Causes → Interventions
Each problem mapped to its root cause and a prioritised intervention playbook. Colour = urgency of resolution.
Problem 1 — Adoption & Value Clarity Gap
Root cause: Stakeholders cannot see the connection between the agent's output and a decision they need to make. Value is abstract, not embedded in a workflow.
Before/after demo: Show one concrete example — "This is what your Wednesday briefing prep looked like before. This is what it looks like now." One workflow, one person, one before/after.
Stakeholder-specific output design: Each audience gets a dedicated view — LT gets executive brief, market leads get country signals, country teams get action flags. Not one generic output.
"SoWhat" mandatory field: Every intelligence output ends with a decision-forcing implication. Removes interpretation burden from the reader.
Problem 2 — AI = Additional Work (Workflow Friction)
Root cause: The agent was designed around what the builder found interesting to build, not around what a user was already doing. It creates a new workflow instead of replacing an existing one.
Replace, don't add: Map every existing workflow the team does manually (e.g. weekly PPT prep, competitive scan, reading industry news). The agent must replace one of these tasks entirely — not add alongside it.
Zero-click delivery: Push outputs to where the user already is — Outlook inbox, Teams channel, PPT template — rather than requiring them to navigate to a new interface.
One-sentence value test: If a user cannot complete "This agent saves me _____ every _____ by automatically _____" — the workflow integration is not clear enough yet.
Problem 3 — Lack of Frontline Buy-in
Root cause: Market teams were informed, not involved. Low ownership leads to passive resistance — not explicit rejection, just non-use.
Co-design sprint (1 session per market): Bring one person from each market team into a 45-min session. Ask: "What do you spend the most time finding or preparing?" Build the next agent to solve that problem specifically.
Name a market champion: One person per market who owns the feedback loop. They are the first user, first critic, and first advocate. Not a bureaucratic role — just the person the hub team calls when outputs need calibrating.
Usage creates ownership: Instead of launching to everyone, launch to one pilot user per market. Let them be seen using it first. Social proof drives adoption faster than communication plans.
Problem 4 — Signal vs Noise
Root cause: Multiple agents running in parallel producing fragmented outputs. No single place where the most important things are visible at once.
Single intelligence surface: All agent outputs flow into one interface — the Intelligence Hub. Power BI dashboard is the canonical view. No output should exist only in email or only in a Teams message.
Maximum 5 signals per cycle: Enforce a hard cap on what gets surfaced per delivery cycle. If the system cannot decide what the top 5 are, the urgency scoring rubric needs work — not the signal volume.
Kill the fragmented outputs: Audit all current agent outputs. Any output that isn't flowing into the central Lists schema and dashboard is creating noise. Consolidate or sunset it.
Problem 5 — Execution Bandwidth Constraint
Root cause: Too many builds in parallel. Each build is too wide in scope. No build is reaching the quality threshold needed to drive adoption before the next one starts.
One agent at a time rule: No new agent starts until the previous one has at least 3 active non-builder users. This is a quality gate, not a timeline.
Pareto scope: For every agent, define the 20% of features that deliver 80% of the value. Build only that. Resist adding features during the first 4 weeks.
Time-box iterations: Each agent gets a 2-week quality sprint after launch. At the end of the sprint: adopt, iterate, or sunset. No agents in indefinite limbo.

Change management — 3 commitments
Stop building new
No new agent starts until the previous one passes the 3-active-user gate. One agent done well is worth more than five agents used by nobody.
Make outputs do work
Every output must arrive in the user's existing workflow — inbox, PPT template, Teams — with a SoWhat statement attached. If it requires a new login or new interface, it will not be used.
Co-design, not broadcast
Market teams become co-owners, not recipients. One champion per market. Their feedback shapes the next iteration cycle. Adoption follows ownership.
